Aqui estão as informações técnicas completas sobre o endpoint de cancelamento de assinatura da Hotmart, conforme a documentação original:

## Parâmetros da requisição

### Path
- **subscriber_code** (string, obrigatório):
  - Código exclusivo de um assinante.

### Body
- **send_mail** (boolean):
  - Indica se deve enviar email de notificação de cancelamento ao comprador.
  - Se true: envia email; se false: não envia email.

### Exemplo de requisição
```bash
curl --location --request POST 'https://developers.hotmart.com/payments/api/v1/subscriptions/:subscriber_code/cancel' \
--header 'Authorization: Bearer :access_token' \
--header 'Content-Type: application/json' \
--data-raw '{
    "send_mail": :send_mail
}'
```

## Retorno
O retorno desta chamada trará os seguintes dados:

- **status** (string): Status atual da assinatura, que pode ser:
    - `ACTIVE` ou `INACTIVE`
- **subscriber_code** (string): Código exclusivo de um assinante, que pode inclusive não ser a mesma pessoa que fez a compra da assinatura.
- **creation_date** (string): Data de criação da assinatura.
- **current_recurrence** (integer): Número da recorrência atual. Exemplo: para assinatura mensal, a primeira recorrência paga é 1, aumentando conforme cada novo ciclo pago.
- **date_last_recurrence** (string): Data do último pagamento.
- **date_next_charge** (string): Data de tentativa do próximo pagamento.
- **due_day** (integer): Dia da cobrança. Para mensal/anual, dia do mês. Para semanal:
    - `1-segunda`, `2-terça-feira`, `3-quarta-feira`, `4-quinta-feira`, `5-sexta-feira`, `6-sábado`, `7-domingo`
- **trial_period** (integer): Dias de período de teste antes da primeira cobrança.
- **interval_type_between_charges** (string): Intervalo de tempo entre cobranças. Valores possíveis:
    - `DAY`, `WEEK`, `MONTH`, `INVOICE`.
- **interval_between_charges** (integer): Número de cobranças conforme o tipo. Ex: mensal + valor 3 = cobrança trimestral.
- **max_charge_cycles** (integer): Quantidade máxima de recorrências. Valor `0` indica "até o cliente cancelar".
- **activation_date** (string): Data de ativação da assinatura.
- **shopper** (object): Usuário responsável pelo pagamento da assinatura, contendo:
    - **email** (string)
    - **phone** (string)
