### Endpoint: Sumário de Assinaturas (Hotmart)

O endpoint de Sumário de Assinaturas fornece uma visão geral do status atual de cada Assinatura, Smart Installment e Smart Recovery, incluindo detalhes da última recorrência e dados para retenção de clientes.

## Tipos de Cobrança Recorrente
1. **Assinatura**: Cobrança recorrente para acesso a um produto, substituindo a cobrança única.
2. **Smart Installments**: Permite ao comprador parcelar o valor do produto em pagamentos mensais, destinado a regiões sem parcelamento convencional. Não permite o cancelamento da assinatura por parte do cliente.
3. **Smart Recovery**: Gera uma nova transação recorrente automaticamente quando a compra é negada por saldo insuficiente, possibilitando tentativas mensais de cobrança para recuperação da venda.

⚠️ **Defasagem dos dados:** Os dados deste endpoint podem ter até 24h de atraso. Para informações em tempo real, utilize o endpoint de [Obter Assinaturas](https://developers.hotmart.com/docs/pt-BR/v1/subscription/get-subscribers/).

---

## Parâmetros da requisição (Query Params)
| Parâmetro               | Tipo     | Descrição                                                                                 |
|-------------------------|----------|-------------------------------------------------------------------------------------------|
| max_results             | integer  | Número máximo de itens por página retornados.                                             |
| page_token              | string   | Cursor de paginação. Use o valor de `next_page_token` para avançar, ou `prev_page_token` para voltar. |
| product_id              | integer  | ID do produto de assinatura (7 dígitos).                                                  |
| subscriber_code         | integer  | Código alfanumérico que identifica uma Assinatura-Assinante.                              |
| accession_date          | integer  | Data de início da assinatura, em ms desde 1970-01-01 00:00:00 UTC. Se omitido, usa data atual -30d. |
| end_accession_date      | integer  | Data de cancelamento da assinatura em ms desde 1970-01-01 00:00:00 UTC.                   |
| date_next_charge        | date     | Data (em ms desde 1970-01-01 00:00:00 UTC) para filtrar assinaturas com próxima cobrança a partir desta data. Se omitido, usa data atual. Para assinaturas canceladas, informa a última data de acesso. |

---

### Exemplo de requisição:
```bash
curl --location --request GET 'https://developers.hotmart.com/payments/api/v1/subscriptions/summary?accession_date=1682910000000&end_accession_date=1696374925000&max_results=500' \
	--header 'Content-Type: application/json' \
	--header 'Authorization: Bearer :access_token'
```

---

## Estrutura de Retorno

- **items**: array contendo objetos com os dados das assinaturas
- **page_info**: objeto com dados de paginação

**page_info contém:**
| Campo              | Tipo    | Descrição                                  |
|--------------------|---------|---------------------------------------------|
| results_per_page   | integer | Quantidade de resultados por página         |
| next_page_token    | string  | Token para buscar a próxima página          |
| prev_page_token    | string  | Token para buscar a página anterior         |

**items contém:**
| Campo                   | Tipo     | Descrição                                                      |
|-------------------------|----------|----------------------------------------------------------------|
| subscriber_code         | string   | Código do assinante                                            |
| subscription_id         | integer  | ID da assinatura                                               |
| status                  | string   | Status da assinatura (ex: "ACTIVE")                            |
| lifetime                | integer  | Tempo de vida da assinatura                                    |
| accession_date          | integer  | Data de início da assinatura (ms)                              |
| end_accession_date      | integer  | Data de cancelamento (ms)                                      |
| trial                   | boolean  | Indica se está em período de teste                             |
| plan                    | object   | Dados do plano:
  - name: string
  - recurrency_period: integer (período de recorrência)
| product                 | object   | Dados do produto:
  - name: string
  - id: integer
| offer                   | object   | Dados da oferta:
  - code: string
| last_recurrency         | object   | Dados da última recorrência:
  - number: integer
  - request_date: integer
  - status: string
  - transaction_number: integer
  - billing_type: string (ex: "SMART_INSTALLMENT")
| unpaid_recurrencies     | array    | Reemissões não pagas:
  - number: integer
  - charge_date: integer
| subscriber              | object   | Dados do comprador:
  - name: string
  - id: integer
  - email: string
