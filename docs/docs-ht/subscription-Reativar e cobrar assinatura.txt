### Reativar Assinatura Inativa — Hotmart API

Este endpoint permite reativar uma assinatura inativa, com a opção de realizar uma nova cobrança para o assinante no momento da reativação.

---

#### Funcionamento e Aprovação
- Para reativar ou reativar com cobrança, o **assinante deve aceitar** a solicitação enviada pelo Produtor.
- O assinante recebe um **e-mail com link válido por 3 dias** para aprovar ou não a reativação.

---

### Parâmetros da Requisição

#### **Path**
- `subscriber_code` (**obrigatório**): Código exclusivo de assinante cuja assinatura será reativada.

#### **Body**
- `charge` (boolean):
  - Indica se deve realizar nova cobrança ao reativar a assinatura.
  - Ao marcar como `true`, uma nova cobrança será gerada. Por padrão, o valor é `false`.
  - A data da cobrança continua a mesma de antes da desativação.

---

#### **Exemplo de Requisição**

```bash
curl --location --request POST 'https://developers.hotmart.com/payments/api/v1/subscriptions/:subscriber_code/reactivate' \
  --header 'Authorization: Bearer :access_token' \
  --header 'Content-Type: application/json' \
  --data-raw '{
      "charge": :charge
  }'
```

---

### Retorno (Response)

- `status` (string): Status atual da assinatura.
  - Pode ser: `INACTIVE`
- `subscriber_code` (string): Código exclusivo do assinante (pode ser diferente do comprador).
- `creation_date` (string): Data de criação da assinatura.
- `interval_between_charges` (integer): Ciclo da assinatura no momento da adesão:
  - `7` (semanal)
  - `30` (mensal)
  - `60` (bimestral)
  - `90` (trimestral)
  - `180` (semestral)
  - `360` (anual)
- `shopper` (object): Dados do usuário pagador.
  - `email`: E-mail do pagador
  - `phone`: Telefone do pagador

#### **Exemplo de Retorno**

```json
{
  "status": "INACTIVE",
  "subscriber_code": "9W2LNSG2",
  "creation_date": "2020-07-20 17:57:42",
  "interval_between_charges": 30,
  "shopper": {
    "email": "shopper@email.com.br",
    "phone": "(31) 988888888"
  }
}
```

---

**Observações:**
- Não omita nenhum parâmetro ou valor listado acima. O processo só é concluído após o aceite do assinante pelo link enviado em até 3 dias.
