### Endpoint: Informações dos Participantes das Vendas (Hotmart)

Este endpoint retorna informações detalhadas sobre os participantes das vendas — como compradores, produtores, afiliados e co-produtores que estejam ligados à produção ou venda do produto. Os dados retornados incluem nome, endereço, telefone e outros detalhes dos participantes.

---

**Parâmetros de Requisição:**

- **transaction** (string): Código único de referência da transação (ex: HP17715690036014).
- **transaction_status** (string): Status da compra. Valores possíveis: `APPROVED`, `BLOCKED`, `CANCELLED`, `CHARGEBACK`, `COMPLETE`, `EXPIRED`, `NO_FUNDS`, `OVERDUE`, `PARTIALLY_REFUNDED`, `PRE_ORDER`, `PRINTED_BILLET`, `PROCESSING_TRANSACTION`, `PROTESTED`, `REFUNDED`, `STARTED`, `UNDER_ANALISYS`, `WAITING_PAYMENT`.
- **max_results** (integer): Número máximo de itens por página.
- **page_token** (string): Cursor de paginação (valor de next_page_token ou prev_page_token).
- **product_id** (long): ID único do produto (7 dígitos).
- **start_date** (long): Data inicial do filtro, em milissegundos desde 1970-01-01 00:00:00 UTC.
- **end_date** (long): Data final do filtro, em milissegundos desde 1970-01-01 00:00:00 UTC.
- **buyer_email** (string): E-mail do comprador para buscar vendas específicas.
- **sales_source** (string): Código SRC usado na página de pagamento para identificação de origem.
- **buyer_name** (string): Nome do comprador.
- **affiliate_name** (string): Nome do afiliado responsável pela venda.
- **commission_as** (string): Como o usuário foi comissionado (`PRODUCER`, `COPRODUCER`, `AFFILIATE`).

Observações importantes sobre os parâmetros:

- Os filtros **transaction** e **transaction_status** devem ser incluídos na query para consultar todos os status desejados. Se não forem informados, serão retornados apenas os status “APPROVED” e “COMPLETE”.

---

**Resposta do Endpoint:**

- **items**: array de objetos, cada um referente a uma venda/transação.
  - **transaction**: código da transação.
  - **product**: objeto com `name` (nome do produto) e `id` (ID do produto).
  - **users**: array contendo objetos para cada participante (ex: produtor, afiliado).
    - **role**: papel do usuário na venda (`PRODUCER`, `AFFILIATE`, `COPRODUCER`).
    - **user**: dados do participante:
      - **ucode**: UUID do usuário
      - **locale**: código do idioma/país
      - **name**: nome
      - **trade_name**: nome fantasia
      - **cellphone**: celular
      - **phone**: telefone
      - **email**: e-mail
      - **documents**: array de documentos (cada um com `type` e `value`, ex: `DOCUMENT`, `CPF`)
      - **address**: objeto com
        - **city**: cidade
        - **state**: estado
        - **country**: país
        - **zip_code**: CEP
        - **address**: logradouro
        - **complement**: complemento
        - **neighborhood**: bairro
        - **number**: número
- **page_info**: objeto contendo as informações de paginação:
  - **total_results**: total de resultados encontrados
  - **next_page_token**: token para próxima página
  - **results_per_page**: quantidade de resultados por página

---

**Exemplo de chamada cURL:**

```bash
curl --location --request GET 'https://developers.hotmart.com/payments/api/v1/sales/users?product_id=123' \
  --header 'Content-Type: application/json' \
  --header 'Authorization: Bearer :access_token''
```
