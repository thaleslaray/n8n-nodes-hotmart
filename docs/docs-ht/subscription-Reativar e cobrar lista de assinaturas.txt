### Endpoint de Reativação de Assinaturas Hotmart

Este endpoint reativa uma lista de assinaturas inativas. Você pode escolher se uma nova cobrança será realizada após o processo. **Atenção:** o assinante deve aceitar a reativação via link enviado por email (válido por 3 dias).

---

## Parâmetros da Requisição

### Body (JSON)
- **subscriber_code** (array de strings, **obrigatório**):
  - Lista de códigos exclusivos de assinantes (`subscriber_code`) cuja assinatura você deseja reativar.
- **charge** (boolean):
  - Indica se deve realizar uma nova cobrança para os compradores ao reativar as assinaturas.
  - Se "true", gera nova cobrança; se "false" (padrão), não gera nova cobrança.
  - A data de cobrança permanece a mesma de antes da desativação.

#### Exemplo de chamada via cURL
```bash
curl --location --request POST 'https://developers.hotmart.com/payments/api/v1/subscriptions/reactivate' \
  --header 'Authorization: Bearer :access_token' \
  --header 'Content-Type: application/json' \
  --data-raw '{
      "subscriber_code": [:subscriber_code],
      "charge": :charge
  }'
```

---

## Retorno da API

- **success_subscriptions** (array de objetos):
  - Lista de assinaturas reativadas com sucesso.
  - Cada objeto contém:
    - **status**
    - **subscriber_code**
    - **creation_date**
    - **interval_between_charges**
    - **shopper** (com campos "email" e "phone")

- **fail_subscriptions** (array de objetos):
  - Lista de assinaturas que **NÃO** puderam ser reativadas, com o motivo do erro.
  - Cada objeto contém:
    - **status**
    - **error**
    - **subscriber_code**
    - **creation_date**
    - **interval_between_charges**
    - **shopper** (com campos "email" e "phone")

#### Exemplo de resposta
```json
{
  "success_subscriptions": [
    {
      "status": "INACTIVE",
      "subscriber_code": "9W2LNSG2",
      "creation_date": "2020-07-20 17:57:42",
      "interval_between_charges": 30,
      "shopper": {
        "email": "subscriber@email.com",
        "phone": "(31) 988888888"
      }
    }
  ],
  "fail_subscriptions": [
    {
      "status": "ACTIVE",
      "error": "SUBSCRIPTION_ALREADY_ACTIVE",
      "subscriber_code": "RGT90XMB",
      "creation_date": "2020-07-08 16:35:57",
      "interval_between_charges": 30,
      "shopper": {
        "email": "subscriber2@email.com.",
        "phone": "(31) 988888888"
      }
    }

```

**Observações importantes:**
- O assinante precisa aceitar a reativação via link por email.
- O link de aceite é válido por 3 dias.
- A reativação pode ser feita com ou sem nova cobrança, conforme o parâmetro "charge".
- Listas enumeradas e parâmetros devem ser enviados exatamente como descritos acima.
