Aqui estão os detalhes completos do evento de primeiro acesso de um aluno a um curso na Hotmart, conforme o conteúdo recuperado:

**Formato dos dados recebidos: JSON**

| Parâmetro                        | Descrição                                                                                                                                                                                                           |
|----------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| hottok (string)                  | Cada conta possui um token único. Ele é enviado no cabeçalho HTTP com o nome `X-HOTMART-HOTTOK`. É fundamental para validar a origem e evitar fraudes. Se precisar trocar, contate o suporte da Hotmart.            |
| id (string)                      | Código único de identificação do evento recebido.                                                                                                                             |
| creation_date (long)             | Data de criação do evento (em milissegundos desde o Unix Epoch - 1970-01-01 00:00:00 UTC).                                                                                   |
| event (string)                   | Nome do evento: neste caso, sempre será `CLUB_FIRST_ACCESS`.                                                                                                                  |
| version (string)                 | Versão do evento: neste caso, sempre será `2.0.0`.                                                                                                                           |
| data (object)                    | Dados relacionados ao evento de primeiro acesso.                                                                                                                              |
| data.product.id (integer)        | Identificador único do produto.                                                                                                                                               |
| data.product.name (string)       | Nome do produto.                                                                                                                                                             |
| data.user.name (string)          | Nome completo do aluno.                                                                                                                                                      |
| data.user.email (string)         | E-mail do aluno.                                                                                                                                                             |

**Exemplo de payload recebido:**

```json
{
  "id": "27b52d28-acf4-448a-bc5f-4ab4bc8dcb35",
  "creation_date": 1632411406874,
  "event": "CLUB_FIRST_ACCESS",
  "version": "2.0.0",
  "data": {
    "product": {
      "id": 3526906,
      "name": "Product Name"
    },
    "user": {
      "name": "Buyer Name",
      "email": "buyer@email.com"
    }
  }
}
```

**Observações Importantes:**
- O campo `hottok` é enviado no cabeçalho HTTP como `X-HOTMART-HOTTOK` e deve ser validado sempre para garantir a autenticidade da notificação.
- Todos os parâmetros devem ser considerados conforme apresentados acima, sem omissões.
- O evento identifica o primeiro acesso de um aluno a um curso (produto) específico.

Caso tenha dúvidas sobre envio ou autenticação desses dados, consulte os tutoriais para Webhook da Hotmart.
