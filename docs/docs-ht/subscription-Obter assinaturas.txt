Aqui estão as informações completas, exatamente como estão na documentação oficial, sobre o endpoint que exibe as informações detalhadas das assinaturas/assinantes do produtor na Hotmart.

## Parâmetros da requisição (Query)

- **max_results** (integer): O número máximo de itens por página que podem ser retornados.
- **page_token** (string): O cursor usado na paginação para navegar pela lista (utilize o valor de next_page_token para acessar a próxima página ou prev_page_token para voltar).
- **product_id** (long): ID do produto de assinatura (7 dígitos).
- **plan** (array containing strings): Nome(s) do(s) plano(s) nos quais o(a) assinante se inscreveu. Aceita múltiplos valores repetindo a chave.
- **plan_id** (long): Identificador único do plano de assinatura.
- **accession_date** (long): Data de início da assinatura (em milissegundos desde 1970-01-01 00:00:00 UTC). Se não informado, considera a data atual menos 30 dias.
- **end_accession_date** (long): Data em que o assinante solicitou o cancelamento da assinatura (em milissegundos desde 1970-01-01 00:00:00 UTC).
- **status** (string): Status atual da assinatura. Valores permitidos: `ACTIVE`, `INACTIVE`, `DELAYED`, `CANCELLED_BY_CUSTOMER`, `CANCELLED_BY_SELLER`, `CANCELLED_BY_ADMIN`, `STARTED`, `OVERDUE`.
- **subscriber_code** (string): Código exclusivo de um assinante.
- **subscriber_email** (string): Email do assinante.
- **transaction** (string): Identificador único de uma transação.
- **trial** (boolean): Indica se a assinatura possui período de teste.
- **cancelation_date** (long): Assinaturas canceladas a partir desta data (milissegundos desde 1970-01-01 00:00:00 UTC). Se não informado, considera a data atual menos 30 dias.
- **end_cancelation_date** (long): Assinaturas canceladas até esta data (milissegundos desde 1970-01-01 00:00:00 UTC). Se não informado, considera a data atual.
- **date_next_charge** (date): Data de próxima tentativa de pagamento (milissegundos desde 1970-01-01 00:00:00 UTC). Para assinaturas canceladas, indica a última data de acesso ao produto. Se não informado, considera a data atual.
- **end_date_next_charge** (date): Filtra assinaturas com tentativa de pagamento até esta data (milissegundos desde 1970-01-01 00:00:00 UTC). Se não informado, considera a data atual.

## Exemplo de requisição

```bash
curl --location --request GET 'https://developers.hotmart.com/payments/api/v1/subscriptions?status=CANCELLED_BY_SELLER&status=ACTIVE' \
	--header 'Content-Type: application/json' \
	--header 'Authorization: Bearer :access_token'
```
