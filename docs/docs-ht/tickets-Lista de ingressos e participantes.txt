## Endpoint: Obter lista de ingressos, compradores e participantes de um evento (Hotmart)

Este endpoint retorna informações detalhadas sobre os ingressos de um determinado evento, incluindo dados dos compradores e dos participantes (quem irá utilizar o ingresso). Apenas ingressos com pagamento confirmado ou convites gratuitos serão listados.

### Parâmetros da Requisição

#### Path
- **event_id** (long, obrigatório):
  - ID do produto (formato Ingresso para Eventos).

#### Query
- **max_results** (integer):
  - Número máximo de itens por página.
- **page_token** (string):
  - Cursor usado para paginação. Use o valor de `next_page_token` ou `prev_page_token` conforme necessidade.
- **buyer_email** (string):
  - E-mail do comprador do ingresso.
- **participant_email** (string):
  - E-mail do participante (quem vai usar o ingresso).
- **last_update** (long):
  - Data da última atualização do ingresso em milissegundos desde 1970-01-01 00:00:00 UTC.
- **id_lot** (long):
  - ID do lote ou categoria do ingresso.
- **ticket_status** (string):
  - `SOLD`: Ingresso com pagamento confirmado.
  - `INVITE`: Convite enviado.
  - `INVITE_CANCELED`: Convite removido.
  - `REFUNDED`: Ingresso reembolsado e inválido.
  - `CHARGEBACK`: Ingresso que sofreu chargeback.
  - `EXCLUDED`: Ingresso excluído.
  - `AVAILABLE`: Ingresso disponível, ainda não atribuído.
  - `RESERVED`: Pagamento pendente ou convite gratuito atribuído.
- **ticket_type** (object):
  - Tipo de ingresso: `PAID`, `FREE`, `ALL`.
- **checkin_status** (object):
  - Status da inscrição: `PENDING`, `PARTIAL`, `CONCLUDED`, `ALL`.
- **id_eticket** (long):
  - ID sequencial do ingresso.
- **ticket_qr_code** (string):
  - Código único, representado no QR Code do ingresso.

#### Exemplo de Requisição

```bash
curl --location 'https://developers.hotmart.com/events/api/v1/:event_id/participants \
--header 'Authorization: Bearer :access_token'
```

### Retorno
- **items** (array de objetos):
  - Lista contendo informações completas do participante, comprador, ingresso (eticket), lote e produto.
- **page_info** (objeto):
  - Detalhes da paginação: `results_per_page`, `next_page_token`, `total_results`.
