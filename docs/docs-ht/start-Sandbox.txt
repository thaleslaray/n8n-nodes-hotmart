O sandbox da Hotmart é um ambiente dedicado para testes de integração durante o desenvolvimento. Ele replica as características do ambiente de produção, permitindo que todos os endpoints dos Hotmart Developers sejam acessados no sandbox. Recomenda-se o uso deste ambiente para identificar eventuais problemas de implementação antes da publicação do sistema.

A autenticação no sandbox funciona exatamente como na API de produção da Hotmart, seguindo as instruções da seção de Autenticação da API (https://developers.hotmart.com/docs/pt-BR/start/app-auth). Todos os dados retornados pelos endpoints do sandbox são fictícios, não impactando sua conta nem dados reais da Hotmart.

- **URL base do Sandbox:** `https://sandbox.hotmart.com/`
- Qualquer endpoint da API pode ser acessado utilizando o caminho original, trocando apenas a base da URL. Por exemplo:
  - Produção: `https://developers.hotmart.com/payments/api/v1/subscriptions`
  - Sandbox: `https://sandbox.hotmart.com/payments/api/v1/subscriptions`

Alguns endpoints possuem múltiplos cenários de resposta, que podem ser testados enviando valores específicos nas requisições. Cada endpoint documentará esses possíveis valores de parâmetros e retornos em sua respectiva seção.

### Retornos de Erro do Sandbox
O sandbox permite testar tanto respostas de sucesso quanto de erro. Embora nem todos os possíveis erros da API estejam disponíveis no sandbox, os principais cenários para validação de integração estão disponíveis. Exemplos de erros comuns:

| Status | Tipo de Erro           | Definição                                                                               |
|--------|------------------------|----------------------------------------------------------------------------------------|
| 404    | not_found              | A URL requisitada não foi encontrada ou está inválida.                                  |
| 405    | method_not_allowed     | O método HTTP usado é conhecido, mas não pode ser aplicado a este endpoint.             |
| 401    | unauthorized_client    | O usuário não possui permissão para realizar a requisição.                              |
| 500    | internal_server_error  | Ocorreu um erro interno inesperado e não foi possível completar a requisição.           |

Erros específicos poderão ser consultados na documentação individual de cada endpoint.
