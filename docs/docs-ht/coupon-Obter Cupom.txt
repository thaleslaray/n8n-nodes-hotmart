### Endpoint para Obter Informações de um Cupom Específico do Produto (Hotmart)

#### Parâmetros da Requisição

**Path:**
- `product_id` (`integer`, obrigatório):
  - Identificador único (ID) do produto vinculado ao cupom (número de 7 dígitos).

**Query:**
- `code` (`string`):
  - Texto que representa um operador como código de cupom.
- `page_token` (`string`):
  - O cursor usado na paginação para acessar diferentes partes da lista de cupons.

#### Exemplo de Requisição

```bash
curl --location --request GET 'https://developers.hotmart.com/products/api/v1/coupon/product/:product_id?code=:code&page_token=:page_token' \
--header 'Content-Type: application/json' \
--header 'Authorization: Bearer :access_token'
```

#### Retorno da API

- `items` (`array` de `object`):
  - Coleção de objetos com as informações dos cupons.
- `page_info` (`object`):
  - Informações de paginação.

##### Descrição dos Campos
- `coupon_code`: Código do cupom.
- `active`: Define se o cupom está ativo (true/false).
- `start_date`: Data de início do cupom (timestamp).
- `discount`: Valor do desconto (ex: 0.1 refere-se a 10%).
- `time_zone`: Objeto com detalhes do fuso horário do cupom.
    - `offset`: Offset do fuso horário.
    - `description`: Descrição textual do fuso horário.
    - `id`: Identificação do fuso horário.
    - `name`: Nome padrão do fuso horário.
- `status`: Status do cupom (ex.: "valid").
- `id`: Identificador único do cupom.

- `page_info`:
    - `next_page_token`: Token para próxima página de resultados.
    - `prev_page_token`: Token para página anterior.
    - `results_per_page`: Quantidade de resultados por página.
